# Protocol Studies

Protocol Studies is the academic field examining how technical protocols — the rules, standards, and specifications that govern how machines and networks communicate — constitute forms of political power, cultural norms, and ontological frameworks. Emerging from software studies, media theory, and STS (Science and Technology Studies), it investigates TCP/IP, HTTP, RSS, Bluetooth, cellular standards, and other protocols not merely as neutral technical specs but as sites of power, exclusion, standardization, and contestation.

## Domain Summary

**What it is:**
Protocol Studies examines the technical protocols underlying digital infrastructure as politico-cultural artifacts. The field emerged from:
- **Galloway's *Protocol* (MIT Press, 2004)** — the foundational text, arguing TCP/IP constitutes distributed control that enables rather than prevents power
- **Wendy Chun's software studies** — code as ideology, "programmed visions"
- **STS traditions** — Bijker, Winner, Latour — showing artifacts have politics
- **Critical infrastructure studies** — Susan Leigh Star, Geoffrey Bowker on classification and standards

**Key insight:** The internet's protocols (TCP/IP, DNS, HTTP) are not neutral technical specifications — they embody specific political philosophies (decentralization, end-to-end design) that produce specific political outcomes (difficult to regulate, surveillance-friendly at certain layers). Similarly, AI protocols (API specs, agent communication frameworks like MCP) embed values in their design.

**Scope:**
- Network protocols (TCP/IP, BGP, DNS)
- Application protocols (HTTP, SMTP, RSS)
- Hardware/radio protocols (Bluetooth, 5G, Wi-Fi standards)
- AI protocols (API specifications, agent communication frameworks)
- Platform protocols (App Store policies as de facto protocol)
- Biometric and surveillance protocols

**Key concepts:**
- *Protocol as control* (Galloway) — control through standards, not through hierarchy
- *Programmed visions* (Chun) — how code shapes what we can imagine
- *Infrastructure inversion* (Star/Bowker) — making visible what's invisible
- *Black-boxing* — protocols hide their politics through technical opacity
- *Interoperability* — who gets to set the terms?
