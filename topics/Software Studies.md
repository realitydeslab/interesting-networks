# Software Studies

Software Studies is an interdisciplinary field that treats software — its code, architectures, protocols, and cultural effects — as a primary object of humanistic and critical inquiry. Rather than studying what software does (computer science) or what it shows (interface studies), Software Studies asks what software *is*: a cultural object, a political structure, an aesthetic medium, a site of power. Emerging from new media studies and digital humanities in the early 2000s, the field crystallized around the MIT Press Software Studies series (est. 2008) and the journal *Computational Culture* (est. 2011). It shares DNA with [[Platform Studies]] (hardware/software affordances), [[Critical Code Studies]] (hermeneutics of source code), and [[Protocol Studies]] (network governance), collectively forming a cluster of "critical computing" fields that refuse to treat technology as neutral infrastructure.

## Domain Summary

Software Studies emerged from several lineages:

1. **Media theory** (McLuhan, Kittler): software as medium, not just tool. Friedrich Kittler's "Es gibt keine Software" (1995) argued software is a fiction concealing hardware's material inscription — a provocation that set the field's tone.
2. **New media studies** (Manovich): *The Language of New Media* (2001) treated software as the cultural logic of digitality — database vs. narrative, numerical representation, automation.
3. **Cultural software studies** (Fuller): Matthew Fuller's *Behind the Blip* (2003) brought cultural studies methods to software artefacts. His edited *Software Studies: A Lexicon* (2008) became the field's founding document.
4. **Platform/code studies** (Bogost/Montfort, Marino): adjacent fields focusing on hardware affordances and code as text.

The field insists on **reading software at multiple layers simultaneously** — interface, code, protocol, infrastructure — and connecting each layer to power, aesthetics, and culture.

## Edge of Chaos Analysis

Software Studies as a named field is **established, not edge-of-chaos** — its founding texts are 15-25 years old. However:
- The theoretical toolkit (Galloway's protocol theory, Fuller's media ecology) is being *applied* to new objects: AI systems, LLMs, agentic software, algorithmic governance
- The field is being re-activated by AI's cultural dominance — what does it mean to study LLM code? Agent protocols? Transformer architectures as cultural forms?
- **For Amber:** The field's methods are a live toolkit for agent ethology — reading agent behavior as software-mediated cultural form

## Key People

- [[Lev Manovich]] — Founding figure. *The Language of New Media* (2001) established software as cultural object. Later founded Software Studies Initiative at UCSD. Now at CUNY. Pioneer of cultural analytics (studying art/culture at scale via computation).
- [[Matthew Fuller]] — UK-based. Edited *Software Studies: A Lexicon* (2008), the field's canonical reference. Professor at Goldsmiths. Brings cultural theory + software art to critical analysis.
- [[Alexander Galloway]] — Protocol theory (*Protocol*, 2004). Argues TCP/IP and internet protocols are forms of distributed control — Deleuzian control societies enacted in code. NYU. Essential for [[Protocol Studies]] connections.
- [[Friedrich Kittler]] — German media theorist. "Es gibt keine Software" (1995) is the field's founding provocation. Media Archaeology approach. Deceased 2011.
- [[Benjamin Bratton]] — *The Stack* (2016): planetary-scale computation as sovereign political structure. Brings Schmittian political theory to platform/software. UCSD, then Strelka Institute.
- [[Noah Wardrip-Fruin]] — Co-founded Software Studies Initiative at UCSD with Manovich and Bratton. Works on expressive processing, electronic literature, and playable media.
- **Mark C. Marino** — Founded Critical Code Studies. Reading code as literature. University of Southern California. [Website](https://markcmarino.com/)
- [[Ian Bogost]] — Co-founded [[Platform Studies]] with Nick Montfort. *Persuasive Games*, *Unit Operations*. Procedural rhetoric: games and software as argument.
- **Nick Montfort** — Co-founded [[Platform Studies]]. MIT. Computational poet. *10 PRINT* (2012) — close reading of a single line of BASIC code.
- **Wendy Hui Kyong Chun** [[Wendy Chun]] — Software as medium of memory and control. *Programmed Visions* (2011). Race and software. Simon Fraser University.
- **Florian Cramer** — Rotterdam. Software art, spam, code poetry. Theoretical overlap with critical code studies.

Key volumes published in the series:

## Related Fields

- [[Platform Studies]] — Bogost & Montfort's series (also MIT Press). Studies the hardware/software platform as shaping creative possibility. Focus: video game platforms, console constraints. Complements Software Studies by going *below* software to silicon.
- [[Critical Code Studies]] — Marino et al. Hermeneutics of source code as text. More microscopic than Software Studies (line-level analysis vs. system-level). Related journal: *Digital Humanities Quarterly*.
- [[Protocol Studies]] — Galloway's contribution. Network protocols as governance. Extends McLuhan/Kittler into TCP/IP, HTTP, DNS. Essential bridge to Amber's protocol interests.
- **Media Archaeology** — Kittler, Parikka, Huhtamo. Excavating discarded media and technical systems. Overlapping methods with Software Studies, more historical focus.
- **Digital Humanities** — Broader field; Software Studies is a critical/theoretical strand within it, often more politicized.
- [[Algorithmic Governance]] — Software Studies meets political theory. How algorithms govern social life.

## Connections to Amber's Work

**1. Agent Ethology ← Software Studies as theoretical grounding**
Agent ethology studies the behavior patterns of artificial agents. Software Studies provides the critical framework for treating agents as *cultural artefacts* — their behavioral repertoires shaped by code, protocol, training data, platform constraints. Manovich's "cultural software" concept maps directly: agents are software that encodes cultural logics. Galloway's protocol theory is essential — agent protocols (API specs, communication standards) are not neutral technical choices but political structures.

**2. Protocol Studies** ← direct lineage
Amber's interest in protocols connects directly to Galloway's *Protocol* (2004) and the Software Studies genealogy. transmediale 2026's "Protocoling" theme is Software Studies territory. Galloway's argument: decentralized networks (internet, agent swarms) are governed through protocol — a distributed but still regulatory form of control.

**3. Cosmotechnics (Yuk Hui) ← Software as techné**
Yuk Hui's cosmotechnics asks how different cultures enact different relationships between cosmos and techné. Software Studies asks similar questions about code as cultural practice. The gap: Software Studies has been largely Western/Eurocentric (McLuhan → Kittler → Galloway). Amber's cosmotechnics perspective could *critique and extend* Software Studies by asking: what would a non-Western software studies look like? How do Chinese internet protocols encode different cosmological assumptions?

**4. More-than-Human Design ← Software as ecological actor**
Software Studies, especially via Fuller's media ecology approach, treats software as part of larger technical ecosystems. Fuller's "software ecology" framing resonates with more-than-human perspectives. What if agents are not tools but ecological participants?

**5. Machine Behavior ← behavioral repertoires as software**
Lex McEwen's and Rahwan's machine behavior research asks: what are the behavioral repertoires of AI systems? This is Software Studies extended to AI. The methodological toolkit — reading code, protocol, platform constraints — applies directly to reading LLM systems as cultural artefacts.

**6. transmediale 2026: "Compassing, Protocoling, Metaphoring"**
"Protocoling" is Software Studies vocabulary. The theme is asking: what orientational practices do computational systems enact? Galloway's protocol theory and Fuller's software ecology provide direct theoretical grounding for any submission here.
